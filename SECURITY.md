# Security Policy

## Supported Versions

This project currently supports security fixes on the latest `main` branch.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security-sensitive problems.

Send a report to `security@zsfd.dev` with:

- a clear description of the issue
- steps to reproduce
- impact assessment
- any proof-of-concept or logs that are safe to share

We will aim to acknowledge new reports within `5 business days`.

## Safe Reporting Notes

- redact API keys, access tokens, cookies, and personal data
- do not include private local transcripts or cached app-data unless strictly necessary
- if the issue involves local token logs, share the smallest reproducible sample

## Out of Scope

The following are generally not treated as security issues for this repository:

- missing support for a third-party client
- inaccurate token counts without a confidentiality, integrity, or availability impact
- feature requests for new adapters or providers
